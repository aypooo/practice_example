자바의 이벤트 처리

이벤트 기반 프로그래밍
이벤트의 종류
사용자의 입력: 마우스 드래그, 마우스 클릭, 키보드  누름등
센서로부터 입력, 네트워크로부터 데이터 송수신
다른 응용 프로그램이나 다른 스레드로부터의 메시지

$이벤트의 발생에 의해 프로그램 흐름이 결정되는 방식
이벤트가 발생하면 이벤트를 처리하는 루틴($이벤트 리스너) 실행
프로그램 내의 어떤 코드가 언제 실행될 지 이벤트 발생에 의해 전적으로 결정

반대 개념: 배치 실행
프로그램의 개발자가 프로그램의 흐름을 결정하는 방식

이벤트 처리 순서
이벤트 발생 - 이벤트 객체 생성- 이벤트 리스너 찾기
- 이벤트 리스너 호출- 이벤트 리스너 실행

이벤트 분배 스레드 
이벤트 소스와 이벤트 종류 결정
적절한 이벤트 객체 생성, 이벤트 리스너를 찾아 호출

이벤트 객체란?
이벤트가 발생할 때, 발생한 이벤트에 관한 정보를 가진 객체
이벤트 리스너에 전달됨

이벤트 리스너란?
이벤트를 처리하는 코드 클래스로 작성
JDK에서 이벤트 리스너 작성을 위한 인터페이스 제공
ActionListener
MouseListener

이벤트 리스너 작성 방법
1.독립 클래스로 작성
2.내부 클래스로 작성
3.익명 클래스로 작성
클래스정의+인스턴스 생성을 한번에 작성


어댑터 클래스
리스너 작성시 추상 메소드들을 모두 구현해야하는 부담되니,
리스너의 모든 메소드가 단순 리턴하도록 구현해 놓은 클래스

key 이벤트와 포커스
키를 '누르고', '떼는'+'떼는(Unicode키의 경우에만)' 순간

키이벤트 받는 조건 
모든 컴포넌트 가능하지만, 현재 포커스(focus)를 가진 컴포넌트
포커스
컴포넌트나 응용프로그램이 키이벤트를 독점하는 권한

유니코드 키
전세계 문자에 대한 코드 체계
문자들에 대해서만 유니코드 값 정의
유니코드 키가 눌러진 경우 호출 순서
keyPressed(),keyTyped(),keyReleased()
유니코드가 아닌 키
문자키가 아닌 다음 키들(제어키)
ex) <Home>,<Up>,<Delete> 등
정의된 유니코드값 없이 키마다 코드값이 정의되어있음
keyPressed(),keyReleased()
